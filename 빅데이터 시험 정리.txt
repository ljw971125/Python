빅데이터 라이프 사이클의 구성 요소는 수집 - 저장/처리 - 분석 - 활용이다.

빅데이터 개발 방법론 : 일반적으로 빅데이터 시스템에 대한 개발 구현과 빅데이터 분석 모델 방법론으로 구성한 빅데이터 서비스 모델을 
개발하기 위한 소프트웨어 공학적 방법론

데이터 라이프 사이클 과정상의 비식별 처리는 개인 프라이버시에 영향을 주지 않으면서 개인 데이터를 사용하기 위한 목적으로 수행된다.
- 데이터 환경 분석서를 작성하는 항목
- 데이터 후보 식별
- 데이터 유형 / 생산주체 / 수집경로
- 데이터 선정 기준
- 후보 데이터 선정 결과
- 정보 보안/ 개인정보 보호 문제
- 제약 사항 및 대안

데이터 표본 추출 방법론 : 데이터 표본 조사 및 추출에 대한 방법을 정의한 빅데이터 서비스 모델을 개발하기 위한 소프트웨어 공학적 방법론
데이터 모델링 방법론 : 데이터 이해 및 준비, 탐색적 및 실증 분석을 통한 데이터의 모델링을 수행하는 빅데이터 서비스 모델을 
개발하기 위한 소프트웨어 공학적 방법론
데이터 품질 관리 방법론 : 서비스 모델의 데이터에 대한 품질 관리 및 성숙도 관리를 하는 빅데이터 서비스 모델을 개발하기 위한 
소프트웨어 공학적 방법론

데이터 라이프 사이클 과정상의 비식별 처리는 (개인) (프라이버시)에 영향을 주지 않으면서 (개인) 데이터를 사용하기 위한 목적으로 수행된다.

데이터 정제 : 빅데이터의 데이터 전·후 처리 방식 중 데이터 이상치, 데이터 결측값 등에 대한 보정 작업을 통해 데이터 신뢰도를 향상시키는 과정
데이터 축소 : 빅데이터의 데이터 전·후 처리 방식 중 대량의 데이터 등으로 인해 크기를 축소하여 빅데이터 분석을 진행하며, 크기를 축소하지
만 동일한 분석 결과가 나올 수 있도록 작업을 진행하는 과정
데이터 변환 : 빅데이터의 데이터 전·후 처리 방식 중 다수의 데이터를 연계통합 작업을 수행하며, 동일한 View에서 볼 수 있도록 통합하는
작업을 수행하는 과정

NCS의 빅데이터 인력에 대한 직무 구분은 (빅데이터 분석)과 DB 엔지니어링이다.
직무 기술서는 해당 직무에 대한 특징과 필요 요건 등을 기록한 서식을 지칭한다.
